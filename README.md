## 启动
因为同源策略的原因，没法直接浏览器发送请求大模型的 api，所以必须转发一下。
使用上，先代理打包好的前端页面：
```
npm run server:dist
```
然后启动 server 转发请求
```
npm run server
```

## worker-tools 产品介绍

### 产品定位
**打工人效率工具箱** — 一款轻量、离线优先的日常工具集合，帮助打工人高效管理任务、记录剪贴板、生成工作日报。

### 目标人群
- 日常需要跟踪任务进度的打工人
- 需要定期写日报、周报的职场人士
- 对工具轻量化和隐私性有要求的用户
- 希望快速生成二维码、分享链接的办公场景

### 如何实现

| 模块 | 技术方案 | 数据存储 |
|------|----------|----------|
| 任务管理 | 原生 HTML/CSS/JS + Tailwind CSS | IndexDB（Dexie.js） |
| 剪贴板 | Clipboard API + paste 事件监听 | IndexDB |
| 日报生成 | Todo 数据 → AI API → Markdown 渲染 | 仅内存，不持久化 |
| 二维码 | qrcodejs（CDN） | 无需存储 |
| AI 代理 | Node.js 轻量转发服务 | 不存储任何数据 |

**架构特点**：
- 前端：单 HTML 文件，零依赖安装
- 后端：极简 Node.js 代理，仅转发请求
- 隐私：所有数据存储在浏览器本地，AI 代理不记录日志

### 项目亮点

1. **单文件交付** — 最终产物仅一个 HTML，可直接双击打开或部署到任意静态服务器

2. **本地优先** — 数据全在浏览器 IndexDB，不上云，隐私无忧

3. **零成本接入 AI** — 用户自配 API Key，服务端仅做 CORS 代理，不产生额外费用

4. **AI 生成日报** — 基于已完成任务自动聚合，按优先级生成一句话工作汇报

5. **现代 UI** — 靛蓝紫渐变 + 卡片式设计，支持 Tab 切换动画和 Toast 提示

6. **多工具集合** — 待办、剪贴板、二维码三合一，减少工具切换成本
